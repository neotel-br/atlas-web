## Endpoints Disponíveis

Base URL: `https://api.neotel.internal/v1`

<Callout type="info">
  Todos os endpoints requerem autenticação via Bearer token, exceto `/auth/login` e `/health`.
</Callout>

---

## Auth

| Método | Endpoint | Descrição |
|--------|----------|-----------|
| `POST` | `/auth/login` | Autenticação com e-mail e senha |
| `POST` | `/auth/refresh` | Renovar access token |
| `POST` | `/auth/logout` | Invalidar tokens |
| `GET` | `/auth/me` | Dados do usuário autenticado |

---

## Documentos

| Método | Endpoint | Descrição |
|--------|----------|-----------|
| `GET` | `/docs` | Listar todos os documentos (paginado) |
| `GET` | `/docs/:id` | Buscar documento por ID |
| `POST` | `/docs` | Criar novo documento |
| `PUT` | `/docs/:id` | Atualizar documento |
| `DELETE` | `/docs/:id` | Remover documento |
| `GET` | `/docs/search?q=` | Busca full-text |

### Query Parameters — GET /docs

| Parâmetro | Tipo | Padrão | Descrição |
|-----------|------|--------|-----------|
| `page` | `number` | `1` | Número da página |
| `limit` | `number` | `20` | Itens por página (máx. 100) |
| `category` | `string` | — | Filtrar por categoria |
| `locale` | `string` | `pt` | Idioma (`pt` ou `en`) |
| `sort` | `string` | `updatedAt` | Campo de ordenação |
| `order` | `string` | `desc` | `asc` ou `desc` |

### Exemplo de Resposta — GET /docs

## Utilitários

| Método | Endpoint | Descrição |
|--------|----------|-----------|
| `GET` | `/health` | Status da API (sem auth) |

## Rate Limiting

| Tipo | Limite |
|------|--------|
| Requisições autenticadas | 100 req/min por usuário |
| Requisições não autenticadas | 10 req/min por IP |
| Endpoint de busca | 20 req/min por usuário |
| Export PDF | 5 req/min por usuário |

Ao exceder o limite, a API retorna `429 Too Many Requests` com o header `Retry-After` indicando quando tentar novamente.
