## Processo de Deploy

<Callout type="warn">
  Deploys em produção são permitidos apenas **terça a quinta**, das 10h às 16h. Nunca na sexta ou vésperas de feriado.
</Callout>

---

## Pré-requisitos

Antes de iniciar qualquer deploy, verifique:

- [ ] Todos os testes passando em CI (`green` no GitHub Actions)
- [ ] PR aprovado por pelo menos **2 revisores**
- [ ] Changelog atualizado em `CHANGELOG.md`
- [ ] Variáveis de ambiente novas adicionadas ao Vault
- [ ] Comunicado no canal `#deploys` no Slack

---

## Deploy em Staging

<Steps>
  <Step>
    Faça merge do PR na branch `develop`:
    ```bash
    git checkout develop
    git merge --no-ff feature/sua-feature
    git push origin develop
    ```
  </Step>
  <Step>
    O pipeline de CD faz deploy automático no ambiente de staging em `staging.neotel.internal`.
    Aguarde a notificação no Slack (`#ci-cd`).
  </Step>
  <Step>
    Valide as mudanças em staging. Use o checklist de QA do ticket correspondente no Jira.
  </Step>
  <Step>
    Marque o ticket como `Ready for Production` no Jira.
  </Step>
</Steps>

---

## Deploy em Produção

<Steps>
  <Step>
    Abra um PR de `develop` → `main` e aguarde aprovação dos revisores.
  </Step>
  <Step>
    Anuncie o deploy no Slack:
    ```
    🚀 Deploy prod iniciando: v2.4.2 — [link do PR]
    Mudanças: [resumo de 1-2 linhas]
    On-call: @seu_nome
    ```
  </Step>
  <Step>
    Faça merge e acompanhe o pipeline no GitHub Actions.
  </Step>
  <Step>
    Monitore os dashboards por **30 minutos** após o deploy:
    - Taxa de erros (< 0,5%)
    - Tempo de resposta P95 (< 800ms)
    - CPU e memória dos containers
  </Step>
  <Step>
    Anuncie a conclusão no Slack com status `✅ Deploy concluído` ou inicie rollback se necessário.
  </Step>
</Steps>

## Rollback

<Callout type="error">
  Em caso de degradação crítica, execute o rollback **imediatamente**. Não espere confirmar a causa raiz.
</Callout>

<Tabs items={['Docker Compose', 'Kubernetes']}>
  <Tab value="Docker Compose">
    ```bash
    # Identifique a última imagem estável
    docker images | grep neodocs-web

    # Rollback via Makefile
    make rollback VERSION=v2.4.1
    ```
  </Tab>
  <Tab value="Kubernetes">
    ```bash
    # Ver histórico de deploys
    kubectl rollout history deployment/neodocs-web

    # Rollback para revisão anterior
    kubectl rollout undo deployment/neodocs-web

    # Ou para revisão específica
    kubectl rollout undo deployment/neodocs-web --to-revision=3
    ```
  </Tab>
</Tabs>

Após o rollback, **abra imediatamente** um incidente no sistema de chamados com severidade P1.
